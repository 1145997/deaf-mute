# 后续开发排期及规划

本文档基于当前项目实际进度编写，目标是把“智能手语翻译工具”从第一期后端骨架逐步推进到可演示、可答辩、可持续迭代的完整版本。

## 当前基线

截至目前，项目已经具备这些基础：
- 后端一期骨架已完成并可启动
- 数据库一期初始化脚本已可执行
- 社区帖子 `post` 模块已可用
- 评论、分类、公告、用户管理、统计、文件上传已打通
- 管理端一期页面已完成联调验收
- 旧 `lostfound` 控制器已默认下线
- 旧 `lostfound` 前端残留与临时路由入口已清理

当前主要缺口：
- 学习模块尚未编码
- 识别配置后台尚未编码
- 客户端识别接入尚未联调
- 识别规则引擎尚未落地

## 总体开发策略

整体建议分四期推进：

1. 一期收口：把现有内容平台做稳，完成前后端语义迁移
2. 二期建模：完成识别配置后台与数据库驱动的规则管理
3. 三期联调：接入客户端 MediaPipe，完成识别链路闭环
4. 四期增强：补学习模块、统计增强、体验优化和答辩材料

原则：
- 先稳定平台，再接识别
- 先保证能演示，再做高级能力
- 先完成后台配置闭环，再做复杂规则树和统计分析

## 第一期：平台一期收口

状态：
- 已完成

### 目标

把原旧业务系统彻底收口成“社区内容平台”一期，保证前后端语义统一、接口统一、数据库统一。

### 范围

- 管理端前端迁移 `lostfound -> post`
- 前端菜单与页面文案迁移
- `post`、`comment`、`category`、`notice`、`statistics` 页面联通
- 清理残留旧接口引用
- 完成一期联调测试

### 主要任务

后端：
- 保持现有一期后端接口稳定
- 补 `post_image` 相关接口细节
- 补必要的异常提示和参数校验
- 清理测试文案，如 `/test/hello`

管理端：
- `src/common/apis/lostfound.ts` 改为 `post.ts`
- `src/pages/lostfound/list.vue` 改为帖子管理页
- `src/pages/lostfound/pending.vue` 改为帖子审核页
- 路由与菜单改名
- 仪表盘口径切换到帖子/评论/用户/公告

测试：
- 管理员登录
- 用户登录
- 分类增删改查
- 帖子发布/列表/详情/我的发布
- 帖子审核通过/驳回
- 评论发布/屏蔽/删除
- 公告管理

### 交付标准

- 管理端不再出现“失物招领”文案
- 默认不再请求 `/api/lostfound/**`
- 一期接口联调通过
- 可完成“登录 -> 发帖 -> 审核 -> 浏览 -> 评论”完整演示链路

### 预计周期

- 3 到 5 天

### 风险点

- 前端页面和旧接口耦合较深
- 管理端表格字段仍可能沿用旧语义字段名

## 第二期：识别配置后台

状态：
- 正式启动

### 目标

把 blind 项目里的硬编码规则抽出来，先做成“后台可配置、数据库可存储”的管理体系。

### 范围

- 基础手势库
- 全局识别配置
- 手势词库
- 单动作管理
- 动作流管理
- 节点配置管理

### 主要任务

数据库：
- 启用 `gesture_library`
- 启用 `recognition_config`
- 启用 `phrase_template`
- 启用 `gesture_flow`
- 启用 `gesture_flow_node`
- 启用 `gesture_flow_output`

后端：
- 新增 `gesture_library` 模块
- 新增 `recognition_config` 模块
- 新增 `phrase_template` 模块
- 新增 `gesture_flow` 模块
- 增加基础枚举：
  - `FlowTypeEnum`
  - `OutputTypeEnum`
  - `ControlActionEnum`

管理端：
- 手势库列表页
- 单动作管理页
- 动作流列表页
- 动作流节点编辑页
- 全局识别配置页
- 短语模板页

初始化数据：
- 从 blind 的基础手势中导入内置手势
- 导入一批默认短语和控制动作
- 给出 2 到 5 条演示规则

### 交付标准

- 管理端可配置基础手势和规则
- 单动作可通过后台新增、修改、启停
- 动作流支持至少线性两步配置
- 数据库存储能替代 blind 中大部分静态映射

### 预计周期

- 5 到 7 天

### 风险点

- 动作流树状编辑器前端实现复杂
- 规则与节点的表单设计需要反复调整

### 建议

这一期先不做复杂流程图，优先用“树表格 + 抽屉表单”实现，确保先可用。

## 第三期：识别链路接入与联调

### 目标

打通“客户端 MediaPipe -> 后端识别 -> 规则命中 -> 返回文本/控制动作”的完整链路。

### 范围

- 识别会话接口
- 识别预测接口
- 基础手势匹配器
- 规则引擎基础版
- 识别记录表

### 主要任务

后端：
- 新增 `recognition/predict`
- 新增 `recognition/session/start`
- 新增 `recognition/session/reset`
- 新增 `recognition_record`
- 实现全局配置 + 节点级覆盖
- 实现单动作规则命中
- 实现基础动作流命中

规则引擎：
- 先支持单动作
- 再支持线性动作流
- 控制动作至少支持：
  - `DELETE_LAST`
  - `CLEAR_ALL`
  - `FINISH_INPUT`

客户端：
- 首页采集 MediaPipe landmarks
- 调用后端识别接口
- 展示文本结果
- 对接简单 TTS

联调测试：
- 单手势 -> 文本
- 两步动作流 -> 短语
- 控制手势 -> 控制动作
- 会话超时与重置

### 交付标准

- 首页识别功能可演示
- 后台规则改动能影响前端识别结果
- 至少支持 5 个基础手势和 2 条动作流演示

### 预计周期

- 7 到 10 天

### 风险点

- 前端 landmarks 上报格式需要与后端模型严格对齐
- 会话状态缓存与超时逻辑容易出 bug
- 识别精度可能受 blind 原型逻辑限制

### 建议

这一期优先做“稳定演示”，不要过早追求复杂树状规则。

### 目标

### 范围

- 学习文章模块
- 学习分类
- 学习页前端
- 识别统计增强
- 识别结果转帖子
- 个人中心增强

### 主要任务

学习模块：
- 学习文章后台管理
- 学习列表和详情接口
- 学习前端展示页

统计：
- 近 7 天识别趋势
- 热门手势命中排行
- 热门短语统计
- 帖子趋势与识别趋势分开展示

功能增强：
- 一键把识别结果写入发布页
- 个人中心增加识别记录
- 增加学习记录、收藏预留字段

答辩准备：
- 系统架构图
- 功能流程图
- 数据库 ER 图
- 演示账号和演示数据

### 交付标准

- 管理端有学习管理入口
- 客户端有学习模块入口
- 首页识别、社区发布、学习浏览三条主线都可演示
- 仪表盘能体现项目特色数据

### 预计周期

- 5 到 7 天
