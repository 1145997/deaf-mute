# 后端 + 安卓端识别消费层接口需求

日期：2026-04-16

提出方：安卓端 uni-app 客户端

对接方：后端主线

## 背景

安卓端当前已进入 `blind -> uni-app` 桥接验证阶段。

现状确认如下：

- `blind` 当前链路为：`MediaPipe H5 -> FastAPI /predict -> 前端句子缓冲与 TTS`
- Spring Boot 主后端当前已具备识别配置后台与公开查询接口
- 但 Spring Boot 当前尚未提供“识别消费层”接口

这意味着：

- 安卓端第一轮桥接目前只能先接 `blind` FastAPI 原型
- 若要进入正式前后端联调，需要后端补齐识别消费层接口

---

## 当前安卓端真实诉求

安卓端不需要后端一次性做完整复杂引擎，但至少需要一套“可替换 `blind /predict`”的最小识别消费接口。

安卓端当前最小需求分两层：

### P0：桥接替换必需

- 获取当前生效识别配置
- 启动识别会话
- 提交 landmarks 做识别
- 重置识别会话

### P1：业务演示增强

- 查询我的识别记录
- 将识别结果沉淀为记录
- 返回可直接展示的输出字段

---

## 建议接口清单

### 1. 获取识别启动配置

建议新增：

- `GET /api/recognition/bootstrap`

用途：

- 安卓端启动识别前一次性拉取本轮所需配置
- 减少首页进入时多接口并发拼装

建议返回：

```json
{
  "activeConfig": {
    "id": 1,
    "configName": "默认识别配置",
    "confidenceMin": 0.75,
    "holdMs": 300,
    "debounceMs": 1200,
    "cooldownMs": 1200,
    "requiredHits": 3,
    "maxIntervalMs": 1500,
    "lockTimeoutMs": 3000,
    "resetOnFail": 1,
    "allowRepeat": 0,
    "gestureOrderJson": "[\"is_v_sign\",\"is_rock_sign\"]"
  },
  "gestureList": [
    {
      "id": 1,
      "gestureCode": "V_SIGN",
      "gestureName": "V 手势",
      "detectionKey": "is_v_sign"
    }
  ],
  "phraseList": [
    {
      "id": 1,
      "phraseCode": "chr_h",
      "phraseText": "我画的",
      "ttsText": "我画的"
    }
  ],
  "flowList": [
    {
      "id": 1,
      "flowCode": "flow_thanks",
      "flowName": "感谢动作流"
    }
  ]
}
```

说明：

- `flowList` 第一版如果暂时不好返回，可先省略
- 但 `activeConfig / gestureList / phraseList` 建议首版直接返回

### 2. 启动识别会话

建议新增：

- `POST /api/recognition/session/start`

用途：

- 安卓端进入首页识别态时创建一次会话
- 后端可为后续锁定状态、动作流、识别记录保留上下文

建议请求：

```json
{
  "clientType": "UNI_APP_ANDROID",
  "sceneCode": "HOME",
  "engineType": "MEDIAPIPE_JS",
  "appVersion": "0.1.0"
}
```

建议返回：

```json
{
  "sessionId": "rec_20260416_0001",
  "startedAt": "2026-04-16T20:00:00",
  "activeConfigId": 1
}
```

### 3. 提交 landmarks 识别

建议新增：

- `POST /api/recognition/predict`

用途：

- 这是安卓端替代 `blind /predict` 的核心接口

建议请求：

```json
{
  "sessionId": "rec_20260416_0001",
  "frameNo": 128,
  "capturedAt": 1713273600000,
  "source": "mediapipe-js",
  "landmarks": [
    { "x": 0.12, "y": 0.45, "z": 0.01 }
  ]
}
```

说明：

- `landmarks` 保持与 `blind` 当前结构一致即可
- 第一版只处理单手 21 点即可

建议返回：

```json
{
  "gesture": "is_v_sign",
  "code": "chr_h",
  "label": "我画的",
  "locked": false,
  "state": "idle",
  "outputType": "TEXT",
  "displayText": "我画的",
  "ttsText": "我画的",
  "controlAction": null,
  "matchedFlowCode": null,
  "matchedNodeCode": null,
  "traceId": "trace_xxx"
}
```

说明：

- 为兼容安卓端当前 bridge 结构，`gesture / code / label / locked / state` 建议必返
- `outputType / displayText / ttsText / controlAction` 建议第一版就加，方便后续直接演进

### 4. 重置识别会话

建议新增：

- `POST /api/recognition/session/reset`

用途：

- 安卓端点击“重置识别”时清空锁定态、动作流上下文、当前识别缓存

建议请求：

```json
{
  "sessionId": "rec_20260416_0001",
  "reason": "USER_RESET"
}
```

建议返回：

```json
{
  "ok": true,
  "message": "session reset"
}
```

### 5. 关闭识别会话

建议新增：

- `POST /api/recognition/session/close`

用途：

- 用户离开首页或识别页时回收会话

### 6. 我的识别记录

建议新增：

- `GET /api/recognition-record/my`

用途：

- 安卓端“个人页”后续展示识别历史

首版建议返回：

- 会话时间
- 识别结果文本
- 命中的动作流 / 短语
- 是否已转为帖子

---

## 字段口径建议

为了减少桥接改造成本，建议后端消费层尽量与 `blind` 现有返回口径对齐：

- `gesture`
- `code`
- `label`
- `locked`
- `state`

同时建议补充新口径：

- `outputType`
- `displayText`
- `ttsText`
- `controlAction`
- `matchedFlowCode`
- `matchedNodeCode`
- `traceId`

---

## 当前后端已有能力可复用点

当前主后端已具备这些基础模型，可直接作为消费层输入来源：

- `gesture_library`
  - `gestureCode / gestureName / detectionKey`
- `recognition_config`
  - `confidenceMin / holdMs / debounceMs / cooldownMs / requiredHits / maxIntervalMs / lockTimeoutMs / gestureOrderJson`
- `phrase_template`
  - `phraseCode / phraseText / ttsText`
- `gesture_flow / gesture_flow_node / gesture_flow_output`
  - 节点阈值
  - 输出类型
  - 控制动作
  - TTS 文本

建议后端消费层优先复用现有配置表，不再另起一套临时硬编码。

---

## 优先级建议

建议按下面顺序推进：

1. `recognition/bootstrap`
2. `recognition/session/start`
3. `recognition/predict`
4. `recognition/session/reset`
5. `recognition-record/my`

---

## 安卓端当前结论

在后端消费层未补齐前：

- 安卓端将继续用 `blind` FastAPI 做第一轮 bridge PoC
- Spring Boot 主后端继续承担配置查询与社区业务

在后端消费层补齐后：

- 安卓端将从 `blind /predict` 切换到 Spring Boot 正式识别接口
- bridge 页面可逐步退化为调试辅助页
